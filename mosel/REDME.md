# 🔬 Mosel Segmentation App

## ✨ Возможности

### 🖼️ Обработка изображений
- ✅ Поддержка форматов: PNG, JPG
- ✅ Работа с изображениями до **200 МБ** и **1 млрд пикселей**
- ✅ Автоматическая обработка любых aspect ratios
- ✅ Auto-padding до кратных 32 пикселей (требование архитектуры)
- ✅ Предобработка: контраст, яркость, цветность, коррекция каналов

### 🤖 Модель
- ✅ Архитектура: **UnetPlusPlus + ResNet50 (MicroNet)**
- ✅ 5 классов (Силикаты, Тальк, Магнетит, Рудная вкрапленность, Другое)
- ✅ Патчевый инференс с `patch_size = 2048`
- ✅ Автозагрузка преобученной модели нга геологических данных с **Hugging Face**

### 📦 Пакетная обработка
- ✅ Загрузка до **50 файлов** одновременно
- ✅ Детальное отображение результатов для каждого файла
- ✅ Сводная таблица по всем файлам

### 💾 Экспорт результатов
- 📄 **CSV** — статистика по 5 классам (пиксели, %)
- 🖼️ **PNG** — цветная карта классов
- 📦 **ZIP** — 5 бинарных масок (по одной на класс)
- 📑 **PDF** — полный отчёт с графиками, таблицами и визуализацией (с поддержкой кириллицы)

## 🖥️ Требования к системе

### Аппаратные требования

| Компонент | Минимум | Рекомендуется |
|-----------|---------|---------------|
| **GPU** | NVIDIA GPU, 12 GB VRAM | NVIDIA RTX 3090/4090 (24 GB) |

| **Диск** | 10 GB свободного места | SSD 20+ GB |
| **CUDA** | 12.1+ | 12.4+ |

### Программные требования

- **Python**: 3.14.x

### 1. Клонирование репозитория

```bash
git clone https://github.com/biritva/mosel.git
cd mosel-segmentation
```
### 2. Создание виртуального окружения

# Linux / macOS
```bash
python3.14 -m venv venv
source venv/bin/activate
```

# Windows
```bash
python -m venv venv
venv\Scripts\activate
```
### 3. Установка зависимостей
```bash
pip install -r requirements.txt
```
### 4. Установка pretrained_microscopy_models
```bash
pip install git+https://github.com/nasa/pretrained-microscopy-models
```
### 5. Запуск приложения
```bash
streamlit run app.py
```
Откройте браузер по адресу: http://localhost:8501

💡 Примечание: При первом запуске модель (~2 ГБ) будет автоматически загружена с Hugging Face.


## 📑 Структура PDF-отчёта

Приложение автоматически генерирует профессиональный PDF-отчёт для каждого обработанного изображения. Отчёт состоит из **4 страниц** и включает полную информацию о результатах сегментации.

### Страница 1: Титульная

| Элемент | Содержание |
|---------|------------|
| 📌 Заголовок | "Отчёт по сегментации микроскопических изображений" |
| 📁 Имя файла | Оригинальное имя загруженного файла |
| 📅 Дата анализа | Дата и время формирования отчёта |
| 📐 Исходный размер | Ширина × высота в пикселях |
| 📏 Размер после обработки | Финальный размер для инференса |
| 🔢 Соотношение сторон | Aspect ratio изображения |
| 🎯 Активные классы | Список из 5 классов сегментации |
| 🔲 Размер патча | Значение `PATCH_SIZE` (2048) |

### Страница 2: Оригинальное изображение

- Полноцветное изображение в исходном виде
- Адаптивные размеры с сохранением пропорций
- Формат: **JPEG** (качество 85) для оптимизации размера PDF
- Максимальная ширина: **1200 px**

### Страница 3: Карта классов (наложение масок)

- Цветная визуализация результатов сегментации
- Наложение всех 5 масок с цветовой кодировкой:
  - ⬛ Чёрный — Силикаты (фон)
  - 🟥 Красный — Тальк
  - 🟩 Зелёный — Магнетит
  - 🟦 Синий — Рудная вкрапленность
  - 🟨 Жёлтый — Другое
- Соответствует размеру оригинального изображения

## ⚙️ Настройки предобработки

Перед подачей изображения в модель, приложение применяет цепочку преобразований для улучшения качества сегментации. Все настройки доступны в **боковой панели** интерфейса.
